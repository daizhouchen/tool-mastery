# Data schema for tool-mastery sites

The `render.py` script consumes a single JSON file with the following shape. Omitted fields are simply skipped in the output — you do not need placeholders.

## Top level

```json
{
  "lang": "en",
  "title": "Browser tab title",
  "description": "Meta description for sharing",
  "kicker": "TOOL MASTERY",
  "heroTitle": "One-line headline that tells the user what this page answers",
  "heroSubtitle": "A 1–2 sentence paragraph expanding the headline",
  "heroMeta": [
    { "label": "Tools", "value": "3" },
    { "label": "Reading time", "value": "~8 min" }
  ],
  "tools": [ /* see Tool */ ],
  "combos": [ /* see Combo — may be empty for single-tool pages */ ],
  "footerLeft": "Generated by the tool-mastery skill",
  "footerRight": "Last updated · 2026-04"
}
```

## Tool

A Tool section is about one product. Keep field counts modest — fewer, sharper cards beat a wall of shallow bullets.

```json
{
  "slug": "linear",
  "name": "Linear",
  "badge": "Issue tracker",
  "tagline": "One short sentence capturing the product's promise",
  "whatItIs": "A 2–4 sentence paragraph. What it is, who it's for, why people choose it over the obvious alternative.",
  "coreCapabilities": [
    { "title": "Issues & cycles", "desc": "One tight sentence. Prefer verbs; name the primitive." }
  ],
  "quickStart": [
    {
      "title": "Create a workspace",
      "desc": "Optional supporting line",
      "code": "Optional CLI snippet or URL",
      "lang": "bash"
    }
  ],
  "scenarios": [
    {
      "title": "Ship a bugfix through the cycle",
      "goal": "Resolve a reported bug and land a fix in the current sprint",
      "steps": [
        "File the bug with a Reproduction section",
        "Triage it into the active cycle",
        "Link the GitHub PR so status updates automatically"
      ]
    }
  ],
  "pitfalls": [
    { "label": "Don't", "text": "Create projects for every small effort — use cycles for lightweight planning." }
  ],
  "migratingFrom": {
    "fromTool": "Jira",
    "mapping": [
      { "from": "Epic", "to": "Project", "note": "Multi-cycle efforts rather than quarterly containers." },
      { "from": "Sprint", "to": "Cycle" },
      { "from": "Story Points", "to": "Estimate" }
    ]
  },
  "docsUrl": "https://linear.app/docs",
  "docsLabel": "Linear documentation"
}
```

**Field conventions**

- `slug` — URL-safe, used for the `#anchor` and TOC. Lowercase, hyphens.
- `badge` — one or two words; category, not feature (e.g. `Issue tracker`, `AI coding CLI`, `Source host`).
- `coreCapabilities` — 3 to 4 items, hard cap at 4. These should be the legs of the mental-model sentence in `whatItIs`.
- `quickStart` — the minimum steps to go from zero to "something real". 3 to 6 steps. Default to a single platform; only stack multi-platform variants if the user explicitly asked for cross-platform. Use `code` for copy-pasteable commands; omit `code` for UI steps.
- `scenarios` — 2 to 3 concrete, real workflows the reader will recognize. Each has a `goal` and ordered `steps`.
- `pitfalls` — 2 to 4 items. Each a non-obvious "the tool won't let you do this gracefully" fact — negative affordance, not generic warnings.
- `migratingFrom` — include **only** when the user explicitly said they're moving from a specific tool. See the dedicated section below.

## migratingFrom (optional)

```json
"migratingFrom": {
  "fromTool": "Jira",
  "mapping": [
    { "from": "Epic", "to": "Project", "note": "optional clarification" },
    { "from": "Sprint", "to": "Cycle" }
  ]
}
```

- `fromTool` — the tool the user is leaving. Title-case the product name.
- `mapping` — 3 to 7 entries. Each row aligns a concept the reader already knows (`from`) with its equivalent in the new tool (`to`). The optional `note` calls out a meaningful non-1:1 difference; leave it out when the mapping is clean.
- Keep the rows focused on the *core primitives* the reader will collide with first. Don't enumerate every UI element.

## Combo

Describes how two or more tools work together. Only include combos when the user asked about integration; skip for single-tool pages.

```json
{
  "slug": "claude-github-linear",
  "title": "Claude Code + GitHub + Linear",
  "tagline": "From Linear ticket to merged PR, tracked automatically",
  "overview": "2–4 sentences describing the joint value. Why wire these together instead of using each alone.",
  "ascii": "Linear issue  ──► Claude Code branch  ──►  GitHub PR  ──►  Linear auto-close\n     │                        │                    │\n     └── assignee sync ◄──────┘                    └─ review bot comments back",
  "workflow": [
    {
      "tools": ["Linear", "Claude Code"],
      "title": "Pull the issue into a working branch",
      "desc": "Claude Code reads the Linear issue ID from the branch name and fetches the ticket context.",
      "code": "claude /start LIN-123"
    }
  ],
  "setup": [
    {
      "title": "Install the Linear GitHub app",
      "desc": "Grant access to the repos that should sync",
      "code": "https://linear.app/settings/integrations/github"
    }
  ],
  "pitfalls": [
    { "label": "Gotcha", "text": "Linear only closes the issue when the PR merges to the default branch — not on any merge." }
  ]
}
```

**Field conventions**

- `ascii` — optional plain-text flow diagram. Kept in a monospace block. Keep it under ~10 lines; it is a visual aid, not a spec.
- `workflow` — ordered, numbered steps. Each step says which `tools` are involved (rendered as pills) plus what happens and the command/action.
- `setup` — one-time integration setup (install an app, add a webhook, grant a scope). Optional.

## Minimal example

```json
{
  "title": "Linear in 5 minutes",
  "heroTitle": "Linear, at a glance",
  "heroSubtitle": "Everything you need to file, track, and ship work.",
  "tools": [
    {
      "slug": "linear",
      "name": "Linear",
      "badge": "Issue tracker",
      "tagline": "The issue tracking tool you'll enjoy using",
      "whatItIs": "Linear is an issue tracker built for product teams who want speed and opinion in one tool.",
      "coreCapabilities": [
        { "title": "Cycles", "desc": "Time-boxed sprints that auto-roll incomplete work forward." },
        { "title": "Views", "desc": "Filtered, saveable slices of issues — your personal queue or team triage." }
      ],
      "quickStart": [
        { "title": "Create a workspace", "desc": "linear.app → New workspace" },
        { "title": "Invite the team", "code": "Settings → Members → Invite" }
      ],
      "scenarios": [
        {
          "title": "Run weekly triage",
          "goal": "Keep Inbox clean without meetings",
          "steps": ["Open the Triage view", "Assign or close each item", "Anything lingering → parked cycle"]
        }
      ],
      "pitfalls": [
        { "label": "Don't", "text": "Use Projects for every feature — reserve them for multi-cycle efforts." }
      ],
      "docsUrl": "https://linear.app/docs"
    }
  ]
}
```
